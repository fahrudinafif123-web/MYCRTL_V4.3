# MYCRTL_V4.3

Berikut adalah rincian mendetail mengenai apa yang terjadi pada perangkat keras dan perangkat lunak saat tombol ditekan:1. Tahap Pertama: Membangunkan Sistem (Wake Up)Jika sistem sedang berada dalam mode tidur (isSleep = true), penekanan tombol akan memicu fungsi wakeUpSystem() untuk (p. 36):Menyalakan Layar OLED: Kontras layar dikembalikan ke tingkat terang normal (oled_br_normal) (p. 23).Mematikan Efek Lampu Tidur: Efek berdenyut (breathing pulse) pada LED C dihentikan secara paksa dan LED C dipastikan mati total (pp. 23, 36).Mencatat Log: Sistem mengirimkan pesan pemicu "WAKE_UP_TRIGGER" ke Serial Monitor (p. 23).Memperbarui Waktu Aktivitas: Waktu aktivitas terakhir (lastActivity) diperbarui ke waktu sekarang menggunakan millis() agar sistem tidak langsung tertidur kembali (pp. 23, 34).2. Tahap Kedua: Memulai Proses Otomasi (Automation Process)Setelah sistem terbangun, jika sistem tidak sedang memproses otomasi sebelumnya (isProcessing = false), maka sistem akan masuk ke Step 1 (p. 37). Di sinilah siklus perhitungan waktu paralel dimulai (pp. 37-38):Pencatatan Waktu Mulai: Sistem mengunci waktu saat tombol ditekan ke dalam variabel startProcessMillis (p. 37).Penguncian Mode Sleep: Indikator hitung mundur tidur (remainingSec) dikunci pada nilai maksimal (sleepDuration) agar layar tidak meredup atau tidur selama mesin bekerja (p. 34).Pengiriman Log: Serial monitor akan menerima laporan status "WAKE_AND_START" atau "BUTTON_START" (p. 37).3. Alur Kerja Relay Setelah Tombol DitekanSetelah tombol dilepas, program di dalam loop() akan terus menghitung selisih waktu dari sejak tombol ditekan (cur - startProcessMillis) (pp. 37-38):Saat Waktu Mencapai Time01: Relay A (LED_A) akan menyala (berubah menjadi LOW) (p. 37).Saat Waktu Mencapai Time02: Relay B (LED_B) akan menyala (berubah menjadi LOW) (p. 38).Saat Kedua Waktu Terpenuhi: Begitu waktu berjalan sudah melewati Time01 DAN Time02, sistem otomatis melompat ke Step 3 untuk menahan posisi relay (p. 38).Saat Jeda Akhir (Time03) Selesai: Setelah ditahan selama durasi Time03, Relay A dan Relay B dimatikan bersamaan (HIGH) (p. 38).Penyimpanan Data: Total siklus (Counter1) bertambah 1, durasi total proses dicatat, data langsung disimpan ke memori penyimpanan (LittleFS), dan sistem kembali ke status siaga (p. 38).4. Proteksi Tambahan (Anti-Gagal)Di dalam kode juga terdapat fungsi proteksi tombol:Debounce Filter: Tombol harus ditekan stabil selama minimal 50 milidetik sebelum sistem merespons (p. 36). Hal ini bertujuan agar getaran mekanis kecil pada tombol tidak memicu proses ganda yang tidak disengaja (p. 36).Boot Protection: Jika tombol dalam posisi tertekan sejak alat pertama kali dinyalakan (booting), sistem sengaja mengabaikannya (btnReady = false) sampai tombol tersebut dilepas terlebih dahulu demi keamanan (pp. 28, 36).
